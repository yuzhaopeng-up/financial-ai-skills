# 厅堂智能分流 (Lobby Routing)

## 概述

厅堂智能分流系统，根据客户特征（年龄、办理业务类型、等候时间、VIP标识等）自动生成最优分流方案，提升网点运营效率，减少客户等候时间，优化客户体验。

## 功能特性

- **智能分流**：根据客户画像匹配最优服务窗口（现金柜口/非现金柜口/自助设备/理财室/贵宾室/手机银行指导台）
- **等待估算**：基于队列人数 × 平均办理时长模型估算等候时间
- **厅堂预警**：拥堵等级评估 + 主动预警（绿/黄/橙/红）
- **动线建议**：指引客户最优行进路线，减少厅堂拥堵

## 客户画像

| 维度 | 说明 |
|------|------|
| 年龄 | 影响渠道偏好（老年人偏好人工柜口） |
| 办理业务类型 | 存款/取款/转账/理财/信用卡/开户等 |
| 等候时间 | 已等候时长，触发优先级调整 |
| VIP标识 | 优先级提升、贵宾室/理财室直达 |

## 分流类型

| 类型 | 适用场景 |
|------|---------|
| 现金柜口 | 现金存取、汇款、挂失等必须临柜业务，老年客户优先 |
| 非现金柜口 | 转账、查询、信用卡还款、电子银行签约等 |
| 自助设备 | 年轻客户，简单的查询、转账、缴费 |
| 理财室 | 理财咨询、产品推荐、风险评估（需VIP或预约） |
| 贵宾室 | 私密业务办理、高净值客户专属服务 |
| 手机银行指导台 | 手机银行不会用需要帮助的客户，现场指导 |

## 等待估算模型

```
预计等待时间 = 队列人数 × 单人平均办理时长
```

各类业务基准时长（分钟）：
- 现金存取：5
- 非现金转账：3
- 信用卡：4
- 理财业务：15
- 查询打印：2

## 拥堵预警

| 等级 | 条件 | 建议措施 |
|------|------|---------|
| 🟢 畅通 | 等候 < 5人/窗口 | 正常运营 |
| 🟡 繁忙 | 等候 5-10人/窗口 | 开启弹性窗口 |
| 🟠 拥挤 | 等候 10-15人/窗口 | 启动疏导，引导自助 |
| 🔴 拥堵 | 等候 > 15人/窗口 | 全员上岗，限制非必要业务 |

## 使用方式

```bash
# CLI 方式
python3 scripts/routing_cli.py generate "智能分流 60岁老人 办理存款 等候20分钟 普通客户"

# Python 方式
from lobby_routing import LobbyRoutingEngine
engine = LobbyRoutingEngine()
result = engine.route(age=60, business_type="存款", wait_time=20, vip=False)
print(result)
```

## 输出字段

| 字段 | 说明 |
|------|------|
| recommended_counter | 推荐柜口类型 |
| estimated_wait_minutes | 预计等候时间 |
| wait_queue_count | 当前排队人数 |
| recommended_products | 推荐产品列表 |
| service_priority | 服务优先级（1-5） |
| congestion_level | 拥堵等级 |
| congestion_advice | 拥堵处置建议 |
| movement_suggestion | 动线建议 |
| alternative_routes | 备选分流方案 |
