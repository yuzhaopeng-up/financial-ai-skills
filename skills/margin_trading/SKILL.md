# Margin Trading Monitor（融资融券监控引擎）

## 概述

融资融券监控技能，提供维保比率计算、平仓预警、追保建议、集中度风险分析及最佳减仓方案。

## 核心参数

| 参数 | 默认值 |
|------|--------|
| 预警线（Warning Line） | 130% |
| 平仓线（Liquidation Line） | 110% |
| 单票集中度预警 | 总持仓30%以上 |
| 单票集中度危险 | 总持仓50%以上 |

## 维保比率公式

```
维持担保比例 = 总资产 / 负债 × 100%
```

- **总资产** = 现金 + 持仓市值（含盈亏）
- **负债** = 融资余额 + 融券市值

## 状态等级

| 比率区间 | 状态 | 颜色 | 说明 |
|----------|------|------|------|
| ≥130% | 🟢 正常 | green | 安全区域，无需操作 |
| 110%~130% | 🟡 关注 | yellow | 预警区间，建议追保或减仓 |
| <110% | 🔴 危险 | red | 平仓风险，立即处理 |

## 功能模块

### 1. 维保比率计算（calculate_maintenance_ratio）
- 输入：持仓（股票+数量+成本+现价）、融资额、融资金额
- 输出：当前维保比率、偏离预警线/平仓线的幅度

### 2. 平仓预警（liquidation_warning）
- 触发条件：维保比率 < 预警线130%
- 计算距离平仓线的安全垫
- 预测不同股价下的维保比率变化

### 3. 追保建议（margin_call_recommendation）
- 方案A：追加保证金（金额计算）
- 方案B：减仓方案（按集中度/亏损排序）
- 方案C：组合方案（部分追保+部分减仓）

### 4. 集中度风险分析（concentration_risk）
- 单票持仓占总资产比例
- 分级预警（关注/警告/危险）
- 分散化建议

### 5. 最佳减仓方案（optimal_reduction_plan）
- 按"亏损率+集中度"双维度排序
- 优先减仓：高亏损 × 高集中度
- 输出：建议减仓数量和比例

## CLI 使用

```bash
python scripts/margin_cli.py generate "融资监控 持仓某股票100万 融资50万 成本10元 现价8元"
python scripts/margin_cli.py analyze --total-assets 1000000 --debt 500000 --positions 某股票,100000,10,8
python scripts/margin_cli.py wecom --positions 某股票,100000,10,8 --debt 500000
```

## 企微卡片

生成结构化飞书/企微消息卡片，包含：
- 维保比率仪表盘
- 持仓明细与盈亏
- 预警状态与建议
- 操作按钮（追加保证金/减仓）

## 依赖

- Python 3.8+
- 无外部依赖（纯计算引擎）
