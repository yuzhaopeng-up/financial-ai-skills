# Lobby Queue Skill — 排队预警 + 智能开台

## 概述

**lobby_queue** 是银行/证券/保险等金融机构大堂经理的智能排队预警引擎，基于等候人数×平均办理时长/服务窗口数计算排队指数（0-100），输出预警等级（红/黄/绿）+ 开台建议 + 客户等候预估 + 历史对比分析。

## 核心能力

- 🎯 **排队指数计算**：等候人数 × 平均办理时长 / 服务窗口数，归一化到 0-100
- 🚦 **三级预警**：绿色（0-40）→ 正常 | 黄色（41-70）→ 注意 | 红色（71-100）→ 告急
- 🏧 **智能开台建议**：根据窗口类型（现金区/非现金区/贵宾区）推荐增开窗口类型
- ⏱️ **等候预估**：基于当前队伍预测平均/最长等候时间
- 📊 **历史对比**：与昨日同期/上周同期对比，分析趋势

## 输入格式（CLI）

```
排队预警 非现金区3人等候 等待最长达25分钟
```

支持格式：
- `排队预警 {区域}{人数}人等候 等待最长达{分钟}分钟`
- `排队预警 {区域}{人数}人等候 {分钟}分钟`
- `排队预警 {区域}{人数}人`

## 输出字段

| 字段 | 说明 |
|------|------|
| queue_index | 排队指数（0-100） |
| alert_level | 预警等级（green/yellow/red） |
| suggestion | 开台建议 |
| wait_estimate | 客户等候预估（分钟） |
| history_compare | 历史对比分析 |
| region | 业务区域 |
| waiting_count | 当前等候人数 |
| max_wait | 最长等候时间（分钟） |
| avg_service_time | 平均办理时长（分钟） |
| service_windows | 服务窗口数 |

## 预警阈值

```
0  ──40──  41──70──  71──100
  绿色      黄色      红色
  正常      注意      告急
```

## 适用场景

- 银行大堂经理实时监控
- 证券公司开户排队管理
- 保险公司理赔大厅
- 任何需要窗口服务排队管理的金融机构
