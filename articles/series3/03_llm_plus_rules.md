# 大模型+规则引擎：金融场景的最佳搭档

> 基于《AI赋能银行数字化转型》第5章技术选型。

---

## 一、纯大模型的局限性

某银行尝试用GPT-4做信贷审批，结果：

- **幻觉问题**：模型"编造"了一条不存在的监管规则
- **不可解释**：拒绝贷款，但说不出具体原因
- **成本高昂**：每月API费用10万+
- **延迟高**：平均响应3秒，用户体验差

**结论**：纯大模型不适合金融核心场景。

---

## 二、纯规则引擎的局限性

传统银行的信贷审批系统：

- **僵化**：规则写死，无法处理例外
- **维护难**：1000+条规则，互相冲突
- **覆盖率低**：只能处理80%的常规 case
- **学习能力弱**：无法从历史数据优化

**结论**：纯规则引擎无法应对复杂场景。

## 三、最佳方案：大模型+规则引擎

### 3.1 架构设计

```
┌─────────────────────────────────────────┐
│              用户请求                    │
│         "申请50万住房按揭"               │
└─────────────────────────────────────────┘
                   ↓
┌─────────────────────────────────────────┐
│              意图识别层                  │
│         （轻量级大模型）                  │
│    识别：信贷申请 / 住房按揭 / 50万       │
└─────────────────────────────────────────┘
                   ↓
┌─────────────────────────────────────────┐
│              规则引擎层                  │
│         （硬约束，不可突破）              │
│    ├── 反欺诈规则：黑名单校验             │
│    ├── 合规规则：监管红线                 │
│    └── 业务规则：准入条件                 │
└─────────────────────────────────────────┘
                   ↓
┌─────────────────────────────────────────┐
│              模型决策层                  │
│         （软约束，可解释）                │
│    ├── 信用评分模型                       │
│    ├── 额度测算模型                       │
│    └── 利率定价模型                       │
└─────────────────────────────────────────┘
                   ↓
┌─────────────────────────────────────────┐
│              大模型增强层                │
│         （生成、解释、优化）              │
│    ├── 生成审批意见                       │
│    ├── 解释拒绝原因                       │
│    └── 推荐优化方案                       │
└─────────────────────────────────────────┘
```

### 3.2 分工明确

| 层级 | 职责 | 技术 | 可解释性 |
|------|------|------|---------|
| **规则引擎** | 硬约束，必须满足 | 确定性规则 | 100% |
| **传统模型** | 量化评分，可解释 | 逻辑回归/决策树 | 80% |
| **大模型** | 生成、解释、辅助 | LLM | 60% |

## 四、实战：信贷审批流程

### 4.1 步骤1：规则引擎过滤

```python
# 硬约束检查
def rule_check(application):
    rules = [
        ("黑名单", lambda x: x.id_number not in blacklist),
        ("年龄", lambda x: 18 <= x.age <= 65),
        ("收入", lambda x: x.income >= 5000),
        ("资料完整", lambda x: len(x.documents) >= 4)
    ]
    
    for name, check in rules:
        if not check(application):
            return False, f"规则不通过: {name}"
    
    return True, "规则通过"
```

### 4.2 步骤2：模型评分

```python
# 信用评分（可解释）
def credit_score(application):
    score = 0
    
    # 身份特质（20分）
    score += min(20, application.age * 0.3)
    
    # 信用历史（25分）
    score += 25 if application.overdue_count == 0 else 0
    
    # 履约能力（25分）
    debt_ratio = application.existing_debts / application.income
    score += max(0, 25 - debt_ratio * 50)
    
    return score
```

## 六、实施建议

### 6.1 技术选型

| 组件 | 推荐方案 | 理由 |
|------|---------|------|

## 七、总结

金融AI的最佳实践不是"用大模型替代一切"，而是"各取所长，分层协作"。

- **规则引擎**：守住底线，100%可解释
- **传统模型**：量化评估，80%可解释
- **大模型**：增强体验，辅助决策
