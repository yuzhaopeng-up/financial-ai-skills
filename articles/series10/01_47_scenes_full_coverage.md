# 我用AI给银行做了47个智能场景：从大堂经理到基金经理全覆盖

> **项目背景**：基于开源金融AI技能库，我们团队（Hermes + 远程审阅节点 + 远程分析节点 多Agent协作）在7天内完成了27个新场景开发，将企微端AI助手从25场景扩展到47场景，覆盖银行、证券、保险、基金四大行业。

---

## 一、为什么要做这件事？

银行数字化转型喊了很多年，但一线员工的真实感受是：

- **大堂经理**：每天面对上百个客户，分流靠经验、营销靠口才、安抚靠情商
- **信用卡专员**：审批规则记不住、催收话术千人一面、逾期客户难管理
- **风控合规**：压力测试报告写一周、反欺诈靠事后追溯、合规培训流于形式
- **证券/基金/保险**：投研报告产量低、客户配置靠感觉、理赔审核效率慢

**我们的思路**：把每个岗位的"经验"变成"规则"，把"重复劳动"交给AI，让员工专注高价值服务。

## 三、技术架构：7个Handler如何协同

```
企微消息 → 场景路由 → Handler分发 → 规则引擎/模拟数据 → 格式化输出
```

**核心设计**：

1. **场景注册中心**（scenario_registry.py）：唯一真相源，47个场景统一注册、冲突检测
2. **Handler映射**：每个岗位一个Handler，职责单一、代码独立
3. **规则+模拟**：基于真实业务规则 + 模拟数据生成可演示的响应
4. **渐进式上线**：`online`/`partial`/`planned`三级状态管理

**代码规模**：

- 6个Handler：3,177行Python
- 场景注册表：369行配置
- 路由映射：26行

## 四、3个让我印象深刻的场景

### 1. 客户情绪识别（大堂经理）

输入："客户等了好久，很着急"

输出：
- 情绪类型：焦虑
- 紧急程度：🟡 中
- 安抚话术："非常理解您的心情，我们已经为您加急处理，预计再等5分钟即可办理"
- 处理建议：立即告知等待时间、提供休息区、协调加急

**价值**：把"靠经验安抚"变成"标准化流程"

### 2. 反欺诈实时预警（风控）

输入："反欺诈预警"

输出：
- 今日预警：5笔
- 触发规则：异地大额交易、快进快出、POS集中交易
- 风险等级：🔴 高 / 🟡 中
- 处置建议：立即冻结、电话核实、上报央行

**价值**：从"事后追溯"到"实时拦截"

### 3. 智能核保（保险）

输入："核保 年龄35 BMI24 血压120/80 不吸烟"

输出：
- 健康评分：85/100
- 规则检查：✅ 年龄合规、✅ BMI正常、✅ 血压正常、✅ 无吸烟史
- 核保结论：🟢 标准体承保
- 建议额度：¥45万

**价值**：从"人工审核3天"到"秒级决策"
