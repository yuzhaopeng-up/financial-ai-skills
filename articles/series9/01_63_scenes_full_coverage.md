# 我用AI搭建了63个金融场景：从对公开户到跨境出海的完整智能体生态

> 两个月，63个场景，31个开源Skill，覆盖金融业的每一个角落

## 前言

两个月前，我在知乎写下第一篇文章：《零API费用！我用Python写了一套银行财务AI智能体》。那时只有6个场景。

今天，这个数字变成了**63**。

从大堂经理的客户分流到基金经理的业绩归因，从对公开户材料清单到企业跨境出海方案，从反洗钱可疑交易报告到ESG可持续发展研究——这63个场景构成了一个**完整的金融AI智能体生态**。

## 技术架构：统一设计模式

所有63个场景遵循相同的四层架构：

```
┌─────────────────────────────────────────┐
│  Layer 1: 自然语言输入层                │
│  用户通过企微/CLI输入指令               │
└─────────────────────────────────────────┘
                   ↓
┌─────────────────────────────────────────┐
│  Layer 2: 场景路由引擎                  │
│  关键词匹配 → 参数提取 → 场景确认       │
│  统一由scenario_registry.py管理         │
└─────────────────────────────────────────┘
                   ↓
┌─────────────────────────────────────────┐
│  Layer 3: 核心引擎层                    │
│  31个独立Skill，每个场景一个引擎        │
│  模板 + 规则 + 轻量LLM                  │
└─────────────────────────────────────────┘
                   ↓
┌─────────────────────────────────────────┐
│  Layer 4: 输出格式化层                  │
│  Markdown报告 / 企微卡片 / PPT大纲      │
└─────────────────────────────────────────┘
```

### 关键设计原则

1. **零API费用**：纯Python实现，不依赖外部LLM API
2. **模板驱动**：80%固定模板 + 20%动态填充
3. **可解释性**：每个结论都有依据和计算过程
4. **可扩展性**：新增场景只需3步：开发引擎 → 注册场景 → 添加handler

## 企微端体验

在企业微信中，用户可以通过以下方式使用：

1. **自然语言输入**：`ESG研究 某新能源企业`
2. **快捷指令**：`/研报`、`/画像`、`/话术`
3. **菜单导航**：选择行业 → 岗位 → 场景

**实际使用示例：**

```
用户: 对公开户 科技有限公司 注册资本1000万
Bot: 🏢 **对公开户辅助**
     
     企业类型: 有限责任公司
     注册资本: 1000万元
     
     📋 **所需材料清单：**
     1. 营业执照正副本
     2. 法定代表人身份证
     3. 公司章程
     4. 股东会决议
     5. 公章、财务章、法人章
     ...
```
